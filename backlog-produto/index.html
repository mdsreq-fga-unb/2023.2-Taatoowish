    Requisitos Não Funcionais

    Tipo Descrição
    Eficiência Agendamentos serão lembrados ao usuário no dia anterior e no dia da sessão
    Usabilidade A aplicação deve ser responsiva, mantendo a funcionalidade e coerência estética para diferentes tamanhos de tela
    Confiabilidade A aplicação necessitará de autenticação para usar determinadas funções
    Suportabilidade A aplicação terá suporte aos navegadores Firefox 118, Google Chrome 117, Safari 14, Microsoft Edge 116

    Priorização

    Para definirmos qual seria o backlog do MVP 1 e MVP 2, priorizamos cada um dos Épicos e das Funcionalidades com dois critérios:

    • Valor de negócio

      • 💰: Pouco valor de negócio.
      • 💰💰: Valor de negócio razoável, mas não é decisivo para a realização do produto.
      • 💰💰💰: Muito valor de negóocio.

    • Complexidade técnica

      • 🔨: Baixa complexidade técnica.
      • 🔨🔨: Trabalhoso, ou seja, demanda mais tempo do que habilidade técnica.
      • 🔨🔨🔨: Alta complexidade técnica.

    Priorizamos primeiramente os Épicos, depois priorizamos as features de um mesmo épico
